Free GitHub Pages Hosting & Custom Domain

Detailed walkthrough using a BootstrapMade template, GitHub Pages, and a free InfinityFree domain.

Overview

This guide shows you how I took a FolioOne template from BootstrapMade, pushed it to GitHub, and served it via GitHub Pages while pointing a free custom domain obtained from InfinityFree.

1. What You Need

  • GitHub account (free).
  • Git/GitHub Desktop experience.
  • Domain from a free provider such as InfinityFree.

2. Prepare the Template

Grab the FolioOne template zip from BootstrapMade, extract and tweak HTML/CSS locally. Preview changes by opening files in your browser or using a local HTTP server (python -m http.server).

3. Push to GitHub

Create a new public repo called yourusername.github.io, then:

git init
git add .
git commit -m "Initial site"
git branch -M main
git remote add origin https://github.com/yourusername/yourusername.github.io.git
git push -u origin main

4. Enable GitHub Pages

In Settings → Pages, choose the main branch and root folder. Save and wait for https://yourusername.github.io to appear.

5. Get a Free Domain

Sign up at InfinityFree and grab an available subdomain (e.g., example.rf.gd). Their nameservers will be used below.

6. Configure DNS or Nameservers

Point the domain at GitHub Pages IPs:

185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153

Or use a CNAME record to yourusername.github.io. GitHub automatically provisions HTTPS once DNS resolves.

7. Add the CNAME File

Create CNAME at repo root with your custom domain, commit and push.

8. Verify & Secure

After propagation, visit your domain. HTTPS should be active. Troubleshoot DNS if needed.

Tips

  • Keep the repo public; required for free Pages.
  • Push updates anytime; site auto‑deploys.
  • Project pages work similar with a CNAME file or gh-pages branch.
